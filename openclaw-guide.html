OpenClaw 安装指南

三种安装方式，覆盖 macOS / Windows / Docker，从零到启动一步步来。

🍎macOS — Gateway 原生安装

macOS Node.js ≥ 18 npm
Step 1

检查 & 升级 Node 版本

确保 Node.js 和 npm 版本足够新（Node ≥ 18 推荐）：

node -v
npm -v

如果版本过低，通过 Homebrew 更新：

brew update
brew upgrade node
Step 2

运行安装脚本

curl -fsSL https://openclaw.ai/install.sh | bash
Step 3

引导初始化 (Onboard)

openclaw onboard --install-daemon

引导过程中你需要：

  • 选择 LLM 提供商（Claude / OpenAI）并粘贴你的 API Key
  • 如需远程访问，将 Tailscale 设为 Serve
  • 如果只需 Web UI，跳过 Telegram/WhatsApp（选 "None" 或按 Ctrl+C）
  • 完成 Persona 设置 — 名称（Name）、风格（Vibe）、Emoji
Step 4

启动 Gateway & 打开面板

获取认证 Token 并启动：

openclaw config get gateway.auth.token
openclaw gateway start

打开控制面板：

openclaw dashboard

浏览器会自动打开：

http://127.0.0.1:18789
💡 如果页面空白，运行 openclaw health 和 openclaw status 检查服务状态。

🐳Docker 版本 — 干净隔离部署

macOS (Intel / Apple Silicon) Windows 11 Windows + WSL2 Docker Desktop
Step 0

安装 Docker Desktop

macOS：下载安装 Docker Desktop

Windows：下载安装 Docker Desktop（确保启用 WSL2 后端）

安装完成后验证：

docker --version
💡 如果提示 Docker 未运行，手动启动 Docker Desktop 应用。
Step 1

创建工作目录 & 配置密钥

mkdir openclaw-docker
cd openclaw-docker

创建 .env 文件并写入你的 API Key：

touch .env

编辑 .env，选择一个填入：

# 二选一，不要同时添加两个
OPENAI_API_KEY=your_key_here
# 或者
ANTHROPIC_API_KEY=your_key_here
Step 2

创建 docker-compose.yml

version: "3.9"

services:
  openclaw:
    image: ghcr.io/openclawai/openclaw:latest
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "18789:18789"
    env_file:
      - .env
    volumes:
      - openclaw_data:/root/.openclaw
    stdin_open: true
    tty: true

volumes:
  openclaw_data:

这个配置会：映射面板端口、持久化记忆数据、隔离运行环境、异常自动重启。

Step 3

启动容器

docker compose up -d

查看日志：

docker logs -f openclaw

打开浏览器访问：

http://127.0.0.1:18789
Step 4

容器内引导初始化

进入容器：

docker exec -it openclaw bash

运行引导程序：

openclaw onboard

选择 LLM、完成 Persona 设置后退出容器：

exit
Step 5

安全文件访问（可选）

如果你想让 OpenClaw 访问主机上的特定文件夹，修改 docker-compose.yml 的 volumes：

volumes:
  - ./workspace:/workspace
  - openclaw_data:/root/.openclaw

然后创建目录：

mkdir workspace
⚠️ OpenClaw 只能访问 /workspace 目录，不会触碰你的系统文件。
Step 6

常用控制命令

# 启动
docker compose up -d

# 停止
docker compose down

# 重启
docker compose restart

# 彻底删除（含数据卷）
docker compose down -v

🪟Windows 原生安装

Windows 10 / 11 PowerShell Node.js 22
🚨 重要：不要使用 CMD！请全程使用 PowerShell（以管理员身份运行）。
Step 0

正确安装 Node 22

不要依赖随意安装的 Node，推荐使用 nvm-windows：

nvm install 22
nvm use 22
node -v
Step 1

修复 PowerShell 执行策略

以管理员身份打开 PowerShell，执行：

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

提示确认时按 Y。

Step 2

安装 OpenClaw

iwr https://openclaw.ai/install.ps1 -useb | iex
⚠️ 如果脚本被拦截：右键文件 → 属性 → 勾选"解除锁定"。

验证安装：

openclaw --version

如果提示 "命令未找到"，手动添加到 PATH：

C:\Users\YourName\.openclaw\bin

然后重启 PowerShell。

Step 3

引导初始化

openclaw onboard --install-daemon

粘贴 API Key 时确保干净，不要带多余空格。

Step 4

处理 Windows Defender 警告

后台守护进程可能触发防火墙提示，选择允许：

  • 允许 "专用网络" 访问
  • 允许 "本地访问"
Step 5

启动 Gateway

openclaw gateway start
openclaw dashboard
常见坑

Windows 特有问题排查

❌ WSL 混乱：不要混用 Windows 原生安装和 WSL 的 Node。只选一个环境，不要交叉使用。
❌ 端口 18789 被防火墙拦截：进入 Windows Defender → 防火墙 → 允许应用通过防火墙，放行 OpenClaw。
❌ Path 无法识别：确认 C:\Users\YourName\.openclaw\bin 已加入系统 PATH 环境变量，并重启 PowerShell。
